# 🚀 Быстрый старт: Оптимизация RAG и эмбеддингов

## 📋 Что было сделано

### Проблема
- RAG поиск не находил информацию в документах (например, "Юпитер" в учебнике астрономии)
- Причина: псевдо-эмбеддинг на основе хешей вместо нейросетевой модели

### Решение
1. **Гибридный эмбеддинг**: Облако + локальный fallback
2. **Совместимость**: Одинаковые векторы для облака и локальной модели
3. **Оптимизация RAG**: Снижены пороги, увеличены лимиты
4. **Кэширование**: Экономия API запросов

---

## ⚡ Настройка за 5 минут

### Шаг 1: Получите API ключ HuggingFace (1 мин)

1. Перейдите на https://huggingface.co
2. Зарегистрируйтесь (если нет аккаунта)
3. Settings → Access Tokens → Create New Token
4. Тип: **read**
5. Скопируйте токен (начинается с `hf_...`)

### Шаг 2: Настройте приложение (2 мин)

1. Запустите приложение
2. Откройте настройки (Settings)
3. Найдите раздел **"Embedding Settings"** или **"Настройки эмбеддинга"**
4. Заполните:
   - **Режим**: Hybrid (Гибридный)
   - **API Key**: вставьте ваш токен `hf_...`
   - **Модель**: `all-MiniLM-L6-v2` (уже стоит по умолчанию)
   - **Провайдер**: HuggingFace
5. Нажмите **"Check API"** → должно появиться `✅ Available`
6. Нажмите **"Apply/Применить"**

### Шаг 3: Проверьте работу (2 мин)

1. Загрузите документ (например, учебник по астрономии)
2. Задайте вопрос: **"Что такое Юпитер?"**
3. Ожидаемый результат: развёрнутый ответ о планете

## 🔧 Дополнительные настройки

### Для работы без интернета

Скачайте локальную модель:

```bash
mkdir -p ~/.local/models/embeddings
cd ~/.local/models/embeddings

# Скачать ONNX версию (~90 MB)
wget https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2/resolve/main/onnx/model.onnx
```

В приложении:
1. Embedding Settings → Local
2. Укажите путь к модели
3. Режим: Hybrid (будет работать и без интернета)

### Увеличение кэша

Для экономии API запросов:

1. Embedding Settings → Cache
2. Cache Size: **1000** (по умолчанию 500)
3. Enable Cache: ✅

## 🎯 Как это работает

### Гибридный режим

```
Запрос → Облако (HuggingFace) → ✅ Успех → Кэш → Результат
                      ↓
                   ❌ Ошибка
                      ↓
              Локальная модель → Результат
```

**Преимущества:**
- ⚡ Быстро (облако ~200-500ms)
- 💪 Надёжно (локальный fallback)
- 💰 Экономно (кэш снижает число запросов)
- 🌐 Работает без интернета (с локальной моделью)

## 🐛 Возможные проблемы

### ❌ "HTTP Error: 429" (Rate Limit)

**Причина:** Превышен лимит запросов HuggingFace

**Решение:**
1. Подождите 5-10 минут
2. Увеличьте размер кэша
3. Добавьте API ключ (если ещё не добавлен)
4. Переключитесь на локальную модель временно

### ❌ "Cloud API: Unavailable"

**Причина:** Нет интернета или проблемы с HuggingFace

**Решение:**
1. Проверьте подключение к интернету
2. Попробуйте позже
3. Переключитесь на локальную модель

### ❌ "Информация не найдена"

**Причина:** Документ не проиндексирован или плохой эмбеддинг

**Решение:**
1. Убедитесь, что документ загружен
2. Проверьте, что эмбеддинг работает (статистика в настройках)
3. Снизьте порог схожести до 0.4
