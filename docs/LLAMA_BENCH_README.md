## 🚀 Как использовать

### 1. Запуск через меню

```
Tools → Llama Bench
```

Или через команду: `open_llama_bench`

### 2. Выбор профилей для сравнения

1. Откройте вкладку **"Compare Profiles"**
2. Отметьте профили для сравнения (Ctrl+Click для множественного выбора)
3. Доступные профили загружаются из папки `profiles/`

### 3. Настройка параметров теста

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| Prompt Tokens | 512 | Количество токенов во входном промпте |
| Generation Tokens | 128 | Количество генерируемых токенов |
| Batch Size | 2048 | Размер батча для обработки |
| Threads | 2 | Количество CPU потоков |
| GPU Layers | 0 | Слои для оффлоада на GPU |
| Repetitions | 3 | Повторений теста для статистики |
| Context Depths | 0,4096,8192 | Глубины контекста для теста |

### 4. Запуск бенчмарка

1. Нажмите **"Start Benchmark"** для одиночного теста
2. Или **"Start Comparison"** для сравнения профилей
3. Прогресс отображается в реальном времени

### 5. Просмотр результатов

Вкладка **"Results"** показывает:
- Prompt TPS (tokens/sec) — скорость обработки входных данных
- Gen TPS (tokens/sec) — скорость генерации
- TTFT (ms) — время до первого токена
- Duration — общая длительность теста
- Status — статус выполнения

**Лучшие значения** подсвечиваются зелёным.

### 6. Экспорт результатов

Кнопки экспорта во вкладке "Results":
- **Export JSON** — для дальнейшего анализа
- **Export CSV** — для импорта в таблицы
- **Export Markdown** — для отчётов
- **Analyze with Model** — анализ результатов моделью (TODO)

## ⚙️ Сборка

```bash
cd llama-gui/build

# Конфигурация (ENABLE_LLAMA_BENCH=ON по умолчанию)
cmake .. -DENABLE_LLAMA_BENCH=ON -DBUILD_TESTS=ON

# Сборка
make -j4

# Запуск
./llama-gui-core
```

### Отключить Llama Bench

```bash
cmake .. -DENABLE_LLAMA_BENCH=OFF
make -j4
```

## 📊 Примеры использования

```bash
# Через CLI llama-bench напрямую
./llama-b7472/llama-bench \
  -m /path/to/model.gguf \
  -p 512 -n 128 \
  -t 2 -ngl 0 \
  -o json
```

1. Tools → Llama Bench
2. Вкладка "Compare Profiles"
3. Выбрать `default.json` и `performance.json`
4. Нажать "Start Comparison"
5. Результаты появятся во вкладке "Results"

## 🔧 Технические детали

### Интеграция с проектом

- **CMake опция:** `ENABLE_LLAMA_BENCH`
- **Пункт меню:** Tools → Llama Bench
- **Команда:** `open_llama_bench`
- **Исполняемый файл:** `../llama-b7472/llama-bench`

### Форматы вывода

llama-bench поддерживает:
- **JSON** (основной) — для парсинга
- **CSV** — для таблиц
- **Markdown** — для отчётов
- **JSONL** — JSON по строкам

### Метрики

| Метрика | Описание |
|---------|----------|
| `prompt_tokens_per_sec` | Скорость обработки входных токенов |
| `generation_tokens_per_sec` | Скорость генерации выходных токенов |
| `prompt_ms_total` | Общее время обработки промпта |
| `generation_ms_total` | Общее время генерации |
| `ttft_ms` | Time To First Token (задержка) |
