# Система конфигурации Llama GUI

## Обзор

Llama GUI использует **многоуровневую систему конфигурации** с чётко определённым приоритетом источников настроек.

## Источники настроек (по приоритету)

### Приоритет 1: Аргументы командной строки
**Уровень:** Высший (переопределяет всё)

Пример:
```bash
./llama-gui-core --debug --url=http://localhost:9090
```

Поддерживаемые аргументы:
- `--debug`, `-d` - режим отладки
- `--url=<server_url>` - URL сервера

---

### Приоритет 2: Профили настроек (profiles/*.json)
**Уровень:** Высокий

**Расположение:** `profiles/<имя_профиля>.json`

**Описание:**
- Полные снапшоты всех настроек приложения
- Формат: JSON
- Автоматически сохраняются при изменении настроек
- Последний изменённый профиль загружается при старте

**Пример:**
```json
{
    "chat": {
        "n_ctx": 4096,
        "max_tokens": 2048,
        "temperature": 0.7
    },
    "batch": {
        "ctx_size": 4096,
        "batch_size": 2048
    },
    "model_loading": {
        "model_path": "/path/to/model.gguf"
    }
}
```

**Команды управления:**
- Сохранить: `Settings::save_profile("name")`
- Загрузить: `Settings::load_profile("name")`
- Список: `Settings::list_profiles()`

---

### Приоритет 3: settings.ini
**Уровень:** Средний (основной файл настроек)

**Расположение:** `settings.ini` (корень приложения)

**Описание:**
- Централизованный файл настроек (аналог php.ini)
- Формат: INI
- Содержит все настройки приложения
- Автоматически синхронизируется с профилями

**Структура:**
```ini
[custom]
model_path = /path/to/model.gguf

[chat]
n_ctx = 4096
max_tokens = 2048
temperature = 0.7

[batch]
ctx_size = 4096
batch_size = 2048

[sampling]
temperature = 0.7
top_p = 0.9

[gpu]
n_gpu_layers = 35

[performance]
enable_logging = true
log_level = Info

[rag]
enable_rag = true
embedding_model_path = /path/to/embed.gguf
```

**Полный список секций:**
- `[custom]` - пользовательские настройки
- `[chat]` - настройки чата
- `[batch]` - настройки пакетной обработки
- `[sampling]` - настройки сэмплирования
- `[model_loading]` - настройки загрузки модели
- `[gpu]` - настройки GPU
- `[cache]` - настройки кэша
- `[rope]` - настройки RoPE
- `[control_vector]` - настройки векторов управления
- `[server]` - настройки сервера
- `[server_runtime]` - настройки выполнения сервера
- `[grammar]` - настройки грамматики
- `[output]` - настройки вывода
- `[performance]` - настройки производительности
- `[display]` - настройки отображения
- `[files]` - настройки файлов
- `[rag]` - настройки RAG

---

### Приоритет 4: Настройки по умолчанию
**Уровень:** Низший (резервный)

**Описание:**
- Встроенные значения по умолчанию
- Используются если нет других источников
- Определяются в конструкторах структур настроек

---

## Синхронизация настроек

### Автоматическая синхронизация при старте

```cpp
Settings settings;
settings.synchronize_at_startup();
```

**Алгоритм:**
1. Проверка наличия профилей
2. Если профили есть → загрузка последнего изменённого
3. Если профилей нет, но есть settings.ini → загрузка из INI
4. Если ничего нет → настройки по умолчанию

### Синхронизация связанных параметров

**Пары синхронизируемых параметров:**

| Параметр 1 | Параметр 2 | Приоритет | Описание |
|------------|------------|-----------|----------|
| `chat.n_ctx` | `batch.ctx_size` | n_ctx | Размер контекста |
| `chat.max_tokens` | `output.n_predict` | max_tokens | Максимальное количество токенов |

**Методы синхронизации:**
- `sync_ctx_size()` - синхронизация размера контекста
- `sync_max_tokens()` - синхронизация максимального количества токенов

**Пример:**
```cpp
// При загрузке профиля
settings.load_profile("default");
settings.sync_ctx_size();      // batch.ctx_size = chat.n_ctx
settings.sync_max_tokens();    // output.n_predict = chat.max_tokens
```

## ConfigManager - Единый центр управления конфигурацией

### Назначение

`ConfigManager` предоставляет унифицированный интерфейс для:
- Инициализации настроек
- Сохранения/загрузки
- Управления профилями
- Резервного копирования
- Валидации

### Использование

```cpp
#include "core/config_manager.h"

using namespace llama_gui::core;

int main() {
    ConfigManager config;
    
    // Инициализация (загружает настройки согласно приоритетам)
    if (!config.initialize()) {
        return 1;
    }
    
    // Доступ к настройкам
    Settings& settings = config.getSettings();
    
    // Сохранение настроек
    config.saveSettings();
    
    // Управление профилями
    config.loadProfile("my_profile");
    config.saveProfile("my_profile");
    
    // Резервное копирование
    std::string backup = config.createBackup();
    config.restoreFromBackup(backup);
    
    return 0;
}
```

### API ConfigManager

| Метод | Описание |
|-------|----------|
| `initialize()` | Инициализация менеджера конфигурации |
| `getSettings()` | Получить доступ к настройкам |
| `saveSettings(sync_profiles)` | Сохранить настройки |
| `loadProfile(name)` | Загрузить профиль |
| `saveProfile(name)` | Сохранить в профиль |
| `listProfiles()` | Список профилей |
| `getCurrentProfileName()` | Текущий профиль |
| `resetToDefaults()` | Сброс к настройкам по умолчанию |
| `validateSettings()` | Валидация настроек |
| `createBackup(path)` | Создать резервную копию |
| `restoreFromBackup(path)` | Восстановить из копии |
| `getDebugInfo()` | Отладочная информация |

## Валидация настроек

### Автоматическая валидация

При инициализации `ConfigManager` автоматически проверяет:
1. Существование путей к конфигам
2. Валидность значений параметров
3. Синхронизацию связанных параметров

### Ручная валидация

```cpp
if (!settings.validate()) {
    std::cerr << settings.get_validation_errors() << std::endl;
}

// Валидация max_tokens
if (!settings.validate_max_tokens()) {
    std::cerr << settings.get_max_tokens_validation_error() << std::endl;
    int recommended = settings.get_recommended_max_tokens();
}
```

## Резервное копирование

### Автоматическое резервное копирование

При изменении настроек рекомендуется создавать резервную копию:

```cpp
std::string backup_path = config.createBackup();
// backup_path = "backups/configs/settings_backup_20260323_143022.ini"
```

### Восстановление

```cpp
if (config.restoreFromBackup(backup_path)) {
    std::cout << "Настройки восстановлены" << std::endl;
}
```

## Рекомендации

### Для пользователей

1. **Используйте профили** для разных сценариев использования
   ```bash
   # Профиль для быстрой генерации
   profiles/fast.json
   
   # Профиль для точной генерации
   profiles/accurate.json
   ```

2. **Регулярно создавайте резервные копии** перед экспериментами

3. **Не редактируйте settings.ini вручную** - используйте UI или API

### Для разработчиков

1. **Всегда используйте ConfigManager** для доступа к настройкам

2. **Не обращайтесь к файлам напрямую** - используйте API:
   ```cpp
   // ✅ Правильно
   config.saveSettings();
   
   // ❌ Неправильно
   settings.save_to_ini("settings.ini");
   ```

3. **Соблюдайте приоритеты** при добавлении новых источников настроек

4. **Документируйте новые параметры** в соответствующих секциях

## Структура файлов конфигурации

```
llama-gui/
├── settings.ini                      # Основной файл настроек
├── profiles/                         # Профили настроек
│   ├── default.json                  # Профиль по умолчанию
│   ├── 1.json                        # Пользовательский профиль 1
│   ├── 2(qween4b).json              # Профиль с именем
│   └── ...
├── backups/configs/                  # Резервные копии настроек
│   ├── settings_backup_20260323_143022.ini
│   └── ...
└── .config/llama-gui/workspaces/    # Рабочие пространства
    └── workspaces_config.json        # Конфигурация workspace
```
