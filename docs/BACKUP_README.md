# 🔄 Автоматический бэкап проекта

## ⚡ Быстрый старт

### Первый запуск

```bash
./scripts/backup.sh "Начальная версия"
```

### Последующие разы

```bash
./scripts/backup.sh "Что изменил"
```

**Всё!** 🎉

---

## 📊 Что происходит

```
┌─────────────────────────────────────────────────────┐
│  Локально (_backups/)                               │
│  └── ТОЛЬКО ПОСЛЕДНИЙ бэкап (~50 MB)                │
│      Предыдущие удаляются автоматически             │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│  Облако (Яндекс.Диск)                               │
│  └── ВСЯ ИСТОРИЯ (до 100 бэкапов)                   │
│      Каждый бэкап сохраняется                       │
└─────────────────────────────────────────────────────┘
```

---

## 📝 Примеры

```bash
# После добавления функции
./scripts/backup.sh "Добавил версию"

# После исправления багов
./scripts/backup.sh "Исправил баги"

# Без комментария
./scripts/backup.sh

# Только локально (не копировать в облако)
./scripts/backup.sh --local-only
```

## 🔄 Восстановление

```bash
# Создать временную директорию
mkdir ~/restore

# Распаковать последний локальный
tar -xzf _backups/llama-gui-*.tar.gz -C ~/restore/

# Или конкретный из облака
tar -xzf ~/Yandex.Disk/Backups/llama-gui/llama-gui-0.1.0-alpha.1-*.tar.gz -C ~/restore/
```

## ⚙️ Настройка

### Изменить путь к облаку

Откройте `scripts/backup.sh`, строка 14:

```bash
CLOUD_DIR="$HOME/Yandex.Disk/Backups/$PROJECT_NAME"
# Измените на свой путь
```

### Изменить лимит облачных бэкапов

Откройте `scripts/backup.sh`, строка ~240:

```bash
if [ "$cloud_backups" -gt 100 ]; then
# Измените 100 на нужное число
```

## ❓ Вопросы

### Сколько места нужно?

- **Локально:** ~50 MB (1 файл)
- **В облаке:** ~5 GB (100 файлов)

### Как часто делать?

После **каждого значительного изменения**.

### Что если забуду?

Ничего страшного! В облаке сохранится предыдущая версия.

### Можно ли отключить облако?

Да:
```bash
./scripts/backup.sh --local-only
```
