# Система автоматического бэкапа проекта

## 📋 Обзор

Система автоматического бэкапа создаёт резервные копии проекта с версионированием:

- **Локально**: Только ПОСЛЕДНИЙ бэкап (экономия места)
- **В облаке (Яндекс.Диск)**: Вся история (до 100 бэкапов)

## 🚀 Быстрый старт

### Первое использование

```bash
# 1. Убедитесь, что скрипт исполняемый
chmod +x scripts/backup.sh

# 2. Запустите бэкап
./scripts/backup.sh "Добавил систему версионирования"
```

## 📖 Использование

### Основные команды

```bash
# Создать бэкап с комментарием
./scripts/backup.sh "Добавил grid snapping"

# Создать бэкап без комментария
./scripts/backup.sh

# Только локально (без копирования в облако)
./scripts/backup.sh --local-only

# Показать список бэкапов
./scripts/backup.sh --list

# Показать справку
./scripts/backup.sh --help
```

## 📁 Структура бэкапов

```
llama-gui/
├── _backups/                      # Локальные бэкапы (ТОЛЬКО ПОСЛЕДНИЙ)
│   └── llama-gui-0.1.0-alpha.2-20260317_120000-alex-pc.tar.gz
│
└── scripts/
    └── backup.sh                  # Скрипт бэкапа
```

### В облаке (Яндекс.Диск)

```
Yandex.Disk/
└── Backups/
    └── llama-gui/
        ├── llama-gui-0.1.0-alpha.2-20260317_120000-alex-pc.tar.gz
        ├── llama-gui-0.1.0-alpha.2-20260316_180000-alex-pc.tar.gz
        ├── llama-gui-0.1.0-alpha.1-20260315_100000-alex-pc.tar.gz
        └── ... (вся история, до 100 бэкапов)
```

## 💡 Стратегия хранения

| Место | Что хранится | Лимит |
|-------|--------------|-------|
| **Локально** | Последний бэкап | 1 файл (~50 MB) |
| **Яндекс.Диск** | Вся история | 100 файлов (~5 GB) |

**Преимущества:**
- ✅ Локально не занимает место
- ✅ В облаке вся история изменений
- ✅ Автоматическая очистка старых бэкапов

## 🔧 Настройка

### Изменение пути к облаку

Откройте `scripts/backup.sh` и измените строку:

```bash
CLOUD_DIR="$HOME/Yandex.Disk/Backups/$PROJECT_NAME"
```

На свой путь:

```bash
# Google Drive
CLOUD_DIR="$HOME/Google\ Drive/Backups/$PROJECT_NAME"

# Dropbox
CLOUD_DIR="$HOME/Dropbox/Backups/$PROJECT_NAME"

# Облако Mail.ru
CLOUD_DIR="$HOME/Mail.ru\ Cloud/Backups/$PROJECT_NAME"

# Свой путь
CLOUD_DIR="/mnt/external_drive/Backups/$PROJECT_NAME"
```

### Изменение лимита облачных бэкапов

В скрипте найдите строку:

```bash
if [ "$cloud_backups" -gt 100 ]; then
```

Измените на нужное число (например, 50):

```bash
if [ "$cloud_backups" -gt 50 ]; then
```

### Исключение файлов из бэкапа

В скрипте найдите секцию `rsync` и добавьте исключения:

```bash
rsync -a \
    --exclude=".git" \
    --exclude="_backups" \
    --exclude="build" \
    --exclude="YOUR_EXCLUDE_HERE" \  # Добавьте свою строку
    ...
```

## 📊 Формат имени файла

```

```

- `llama-gui-0.1.0-alpha.2-20260317_150000-alex-pc-Dobavil_setku.tar.gz`
- `llama-gui-dev-20260318_090000-alex-pc.tar.gz` (без версии)

## 📦 Что входит в бэкап

### ✅ Включается

- Весь исходный код (`.cpp`, `.h`)
- Файлы конфигурации (`CMakeLists.txt`, `settings.ini`)
- Документация
- Скрипты
- Файлы локализации
- Шрифты
- Модели (если есть в проекте)

### ❌ Исключается

- `.git/` — директория Git
- `build/` — директория сборки
- `_backups/` — предыдущие бэкапы
- `*.log` — логи
- `*.swp` — временные файлы редакторов
- `.DS_Store` — macOS метаданные
- `*.pyc`, `__pycache__/` — Python кэш

## 🔄 Автоматизация

### Бэкап перед компиляцией

```bash
# Создайте алиас в ~/.bashrc
alias build='./scripts/backup.sh --local-only && mkdir -p build && cd build && cmake .. && make -j4'

# Теперь просто запускайте:
build
```

### Бэкап по расписанию (cron)

```bash
# Откройте crontab
crontab -e

# Добавьте строку для ежедневного бэкапа в 18:00
0 18 * * * /home/alex/projects/llama-gui/scripts/backup.sh

# Или для еженедельного бэкапа в воскресенье в 20:00
0 20 * * 0 /home/alex/projects/llama-gui/scripts/backup.sh
```

## 🌐 Поддержка других облаков

google-drive-ocamlfuse ~/GoogleDrive

### Dropbox

```bash
# Установите официальный клиент Dropbox
# После установки:

```

### Rclone (универсальный клиент)

```bash
# Установите rclone
sudo apt-get install rclone

# Настройте подключение
rclone config

# Смонтируйте облако
rclone mount yandex: ~/YandexDisk --daemon

# В backup.sh укажите:
CLOUD_DIR="$HOME/YandexDisk/Backups/$PROJECT_NAME"
```

## 📊 Статистика бэкапов

```bash
# Распаковать и посмотреть BACKUP_INFO.txt
tar -xzf _backups/llama-gui-*.tar.gz
cat llama-gui/BACKUP_INFO.txt
```

### Размер бэкапов

```bash
# Показать размер локального бэкапа
du -sh _backups/

# Показать размер облачных бэкапов
du -sh ~/Yandex.Disk/Backups/llama-gui/
```

## 🛡️ Восстановление из бэкапа

### Из локального бэкапа

```bash
# 1. Создайте директорию для восстановления
mkdir ~/restore

# 2. Распакуйте бэкап
tar -xzf _backups/llama-gui-*.tar.gz -C ~/restore/

# 3. Проверьте файлы
ls -la ~/restore/llama-gui/

# 4. Скопируйте нужные файлы
cp -r ~/restore/llama-gui/src/* /path/to/project/
```

### Из облачного бэкапа (конкретной даты)

```bash
# 1. Посмотрите доступные бэкапы
ls -lh ~/Yandex.Disk/Backups/llama-gui/

# 2. Выберите нужную дату и распакуйте
tar -xzf ~/Yandex.Disk/Backups/llama-gui/llama-gui-0.1.0-alpha.1-*.tar.gz -C ~/restore/
```

## 💡 Советы

1. **Регулярность**: Делайте бэкап после каждого значительного изменения
2. **Комментарии**: Всегда указывайте комментарий — это поможет найти нужный бэкап в облаке
3. **Проверка**: Периодически проверяйте, что бэкапы восстанавливаются
4. **Облако**: Убедитесь, что Яндекс.Диск синхронизируется корректно

## 🆘 Решение проблем

### Ошибка: "Яндекс.Диск не найден"

**Проблема:** Путь к облаку не существует

**Решение:**
```bash
# Проверьте, смонтирован ли Яндекс.Диск
ls ~/Yandex.Disk

# Если нет — запустите клиент
yandex-disk start

# Или измените путь в backup.sh
```

### Ошибка: "Недостаточно места"

**Проблема:** Закончилось место на диске

**Решение:**
```bash
# Проверьте размер локального бэкапа
du -sh _backups/

# Должен быть только 1 файл (~50 MB)
# Если больше — скрипт не сработал
ls -lh _backups/
```

### Бэкап слишком большой

**Проблема:** Архив занимает много места

**Решение:**
```bash
# Добавьте дополнительные исключения в backup.sh
--exclude="models/"
--exclude="deps/"
--exclude="external/"
```

### Облако не синхронизируется

**Проблема:** Файлы не появляются в Яндекс.Диске

**Решение:**
```bash
# Проверьте статус клиента
yandex-disk status

# Перезапустите клиент
yandex-disk stop
yandex-disk start

# Проверьте путь в скрипте
grep CLOUD_DIR scripts/backup.sh
```
